# 智保灵犀·多端自适应 —— 产品技术方案

> 目标：`web/static/index.html` 在 PC、Pad、手机浏览器中自动适配，根据设备类型呈现合适布局与交互。

---

## 一、产品需求

### 1.1 用户场景

| 场景 | 设备 | 典型分辨率 | 核心诉求 |
|------|------|------------|----------|
| 办公/桌面 | PC | ≥1024px | 保持现有左右分栏，侧栏常显 |
| 便携办公/学习 | Pad | 768px～1023px | 侧栏可收起，主区充分利用 |
| 碎片化咨询 | 手机 | ＜768px | 全屏对话，侧栏抽屉式，大按钮易点 |

### 1.2 验收目标

| 设备 | 预期表现 |
|------|----------|
| PC | 与现有一致，布局、字号、间距保持 |
| Pad | 侧栏可折叠；主区宽度自适应；输入区舒适 |
| 手机 | 单栏对话；侧栏抽屉；欢迎语、气泡、按钮适配小屏；防止误触 |

### 1.3 设计原则

1. **移动优先**：以 320px 为最小宽度基线，确保基本可用
2. **触控友好**：可点击区域≥44×44px，间距≥8px
3. **内容优先**：小屏下优先展示对话内容，侧栏可隐藏
4. **渐进增强**：大屏保留并优化现有布局

## 二、现状问题分析

### 2.1 当前布局结构

```
┌─────────────────────────────────────────────────────┐
│  aside (w-64 固定)  │  main (flex-1)                 │
│  - 新对话按钮        │  - header (模型选择)            │
│  - 最近记录         │  - chat-content (对话区)         │
│  - 主题切换         │  - input-area (输入区)           │
│  - 用户信息         │                                 │
└─────────────────────────────────────────────────────┘
```

### 2.2 小屏暴露的问题

| 问题 | 表现 | 影响 |
|------|------|------|
| 侧栏固定 256px | 小屏占用过半宽度 | 对话区过窄，体验差 |
| 无侧栏收起 | 无法隐藏 | 无法专注对话 |
| 欢迎语字号大 | text-5xl 在小屏溢出 | 首屏观感差 |
| 输入区固定 | p-6、max-w-4xl 占满 | 输入框偏小、按钮拥挤 |
| 触控区域小 | 部分按钮 p-3、text-xs | 易误触、不易点 |
| 表格/长内容 | 无横向滚动 | 内容溢出或换行错乱 |

### 2.3 已有基础

- `viewport` 已配置 `width=device-width, initial-scale=1.0`
- 使用 Tailwind CSS，具备响应式类（sm/md/lg/xl）
- 无固定 px 宽度的关键内容区（除 aside）

---

## 三、技术方案

### 3.1 断点与策略

采用 Tailwind 默认断点（与项目一致）：

| 断点 | 宽度 | 策略 |
|------|------|------|
| 默认 | ＜640px | 手机：单栏 + 抽屉侧栏 |
| sm | ≥640px | 小平板：可适度两栏 |
| md | ≥768px | 平板：侧栏可折叠，主区扩展 |
| lg | ≥1024px | PC：保持当前布局 |

### 3.2 布局方案

#### 3.2.1 侧栏：响应式 + 抽屉

| 断点 | 侧栏表现 |
|------|----------|
| ＜1024px | 默认隐藏，通过「菜单」按钮打开（从左侧滑入的抽屉） |
| ≥1024px | 常显，宽度 w-64 |

**实现思路**：

- 增加「菜单」按钮（仅 lg 以下显示），点击切换侧栏显隐
- 侧栏：`fixed` + `z-40`，`transform translateX` 控制滑入滑出
- 遮罩层：侧栏打开时覆盖主区，点击遮罩关闭
- 过渡：`transition-transform duration-300`

#### 3.2.2 主区与对话区

- 主区：始终 `flex-1 min-w-0`，避免 overflow
- 对话气泡：`max-w-3xl` → `max-w-[95%] sm:max-w-3xl`，小屏略收窄
- 输入区容器：`p-6` → `p-4 sm:p-6`，`max-w-4xl` 保持

#### 3.2.3 欢迎区

- 标题：`text-5xl` → `text-3xl sm:text-4xl lg:text-5xl`
- 副标题：`text-lg` → `text-base sm:text-lg`
- 上边距：`mt-32` → `mt-16 sm:mt-24 lg:mt-32`

#### 3.2.4 输入区

- 整体：`p-4` → `p-3 sm:p-4`
- 发送按钮：`px-8 py-2.5` → `px-6 py-3 sm:px-8 sm:py-2.5`，保证触控面积
- 附件按钮：`p-3` 保持（约 44px）

#### 3.2.5 Header

- 模型选择：`text-xl` → `text-base sm:text-xl`
- 小屏可考虑折叠为图标 + 下拉，或保持当前 select

#### 3.2.6 认证页

- 容器：`p-10` → `p-6 sm:p-10`
- 标题：`text-3xl` → `text-2xl sm:text-3xl`
- 保持 `max-w-md`，小屏留边距

### 3.3 表格与长内容

- 答案区表格：`overflow-x-auto` 包裹，允许横向滚动
- 代码块、长 URL：`word-break: break-all` 或 `overflow-wrap: break-word`

### 3.4 安全区域

- 为刘海屏预留：`padding-top: env(safe-area-inset-top)` 等（可选，按需启用）
- 底部输入区：`padding-bottom: env(safe-area-inset-bottom)`（可选）

## 四、改动清单

### 4.1 HTML 结构调整

| 位置 | 改动 |
|------|------|
| 主应用容器 | 增加 `overflow-x-hidden`，避免横向滚动条 |
| 侧栏 aside | 增加 `lg:flex`、`hidden`、`fixed`、`inset-y-0`、`left-0`、`z-40`；lg 以上为 `lg:relative`、`lg:flex` |
| 遮罩层 | 新增 `#sidebar-overlay`，侧栏展开时显示，点击关闭侧栏 |
| 菜单按钮 | 新增 `#btn-toggle-sidebar`，lg 以下显示，lg 隐藏 |
| Header | 左侧增加菜单按钮占位，保证标题/选择器居中 |

### 4.2 样式类调整

| 元素 | 原类 | 调整后 |
|------|------|--------|
| 欢迎标题 | `text-5xl` | `text-3xl sm:text-4xl lg:text-5xl` |
| 欢迎副标题 | `text-lg` | `text-base sm:text-lg` |
| 欢迎区上边距 | `mt-32` | `mt-16 sm:mt-24 lg:mt-32` |
| 对话气泡容器 | `max-w-3xl` | `w-full max-w-[95%] sm:max-w-3xl` |
| chat-content padding | `p-6` | `p-4 sm:p-6` |
| 输入区容器 | `p-6` | `p-4 sm:p-6` |
| 输入区内边距 | `p-4` | `p-3 sm:p-4` |
| 认证卡 | `p-10` | `p-6 sm:p-10` |
| 认证标题 | `text-3xl` | `text-2xl sm:text-3xl` |
| ai-answer-content table | （无） | 父级加 `overflow-x-auto` |

### 4.3 JavaScript 新增

| 功能 | 说明 |
|------|------|
| 侧栏显隐 | `btn-toggle-sidebar` 点击切换 class（如 `-translate-x-full` / `translate-x-0`） |
| 遮罩关闭 | `sidebar-overlay` 点击时关闭侧栏 |
| 响应式关闭 | `resize` 或 `matchMedia`：宽度≥1024px 时自动关闭遮罩、展开侧栏 |
| 历史点击 | 小屏下点击历史项后自动关闭侧栏 |

## 五、实现步骤建议

### 第一阶段：基础响应式

1. 调整欢迎区、对话区、输入区、认证区的 Tailwind 响应式类
2. 为表格等长内容增加横向滚动
3. 在真机与不同宽度下验证布局

### 第二阶段：侧栏抽屉

1. 增加菜单按钮、遮罩层、侧栏的 HTML
2. 实现侧栏显隐逻辑
3. 处理 resize / matchMedia 的自动适配
4. 历史点击后关闭侧栏（可选）

### 第三阶段：细节优化

1. 触控区域、间距复查
2. 安全区域（刘海屏等）按需配置
3. 横屏、折叠屏等特殊设备简单测试

---

## 六、验收标准

| 设备类型 | 验证项 |
|----------|--------|
| PC (≥1024px) | 布局与现有一致；无功能退化 |
| Pad (768～1023px) | 侧栏可折叠；主区宽度合理；输入区可用 |
| 手机 (＜768px) | 单栏布局；侧栏抽屉；按钮易点；无横向溢出；欢迎语不溢出 |
| 通用 | 登录/注册、发送、流式回复、历史点击等核心流程正常 |

---

## 七、附录：Tailwind 响应式类速查

```
默认: 无前缀 (< 640px)
sm:  ≥640px
md:  ≥768px
lg:  ≥1024px
xl:  ≥1280px

隐藏/显示: hidden / block / flex
lg:block  → 仅 lg 以上显示
lg:hidden → 仅 lg 以下显示
```
