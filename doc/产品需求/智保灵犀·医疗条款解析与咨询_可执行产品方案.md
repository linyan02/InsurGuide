# 智保灵犀·医疗条款解析与咨询 —— 可执行产品方案

> 基于项目整体架构的医疗类保险条款咨询与解析能力设计

> 文档版本：v1.0 | 最后更新：2025-02-26

## 一、需求背景与目标

### 1.1 核心痛点

| 用户痛点 | 描述 |
|---------|------|
| 条款难懂 | 医疗险、重疾险等条款专业晦涩，普通用户难以理解 |
| 咨询不便 | 手头有合同/条款 PDF，想快速问「这段什么意思」「能赔吗」但缺乏便捷入口 |
| 输入门槛高 | 仅支持文字输入，粘贴长条款不便；无法直接上传 PDF/Word |

### 1.2 功能定位

**医疗条款解析与咨询（Medical Insurance Clause Parser & Consultant）**：用户通过**文字输入**或**文件上传**（PDF/Word/TXT）提供医疗类保险条款，即可基于该条款进行多轮咨询、解读与答疑。

### 1.3 产品目标

1. **输入多样化**：支持文本粘贴、文件上传（PDF、Word、TXT 等）
2. **条款感知**：识别用户是否携带条款上下文，优先基于该条款回答
3. **多轮咨询**：在「当前条款」下持续提问，无需重复输入
4. **知识增强**：结合平台保险知识库，对条款做专业解读与补充说明

## 二、现状分析

### 2.1 当前架构与可复用能力

```
用户提问 ──► Redis 取上下文 ──► 意图识别 ──► 问题改写 ──► RAGflow 检索 ──► 答案生成 ──► 合规
                   │                    │
                   │                    └── 新增 INTENT_CLAUSE_PARSE 医疗条款解析
                   └── 上下文：可携带「当前条款内容」或「用户知识库 ID」
```

**可复用模块**：

| 模块 | 文件 | 复用方式 |
|------|------|----------|
| 意图识别 | `app/intent.py` | 新增 `INTENT_CLAUSE_PARSE` 意图 |
| RAGflow 检索 | `app/ragflow_client.py` | 支持多库检索，可叠加用户条款知识库 |
| 答案生成 | `app/answer_engine.py` | 专用 Prompt 模板（条款 + 知识库） |
| 上下文存储 | `core/redis_store.py` | 存储当前条款 ID / 文本引用 |
| 合规校验 | `app/compliance.py` | 沿用现有合规逻辑 |

**RAGflow 能力**：

- 支持创建 Dataset、上传文档（`POST /api/v1/datasets/{dataset_id}/documents`）
- 支持 PDF、Word、TXT 等格式解析
- 检索接口支持 `dataset_ids` 数组，可同时检索多个知识库

**当前缺口**：

- 无用户级/会话级文档上传接口
- 无「当前条款」上下文携带机制
- 无条款解析专用意图与流程

### 2.2 与现有流程的差异

| 维度 | 现有通用咨询 | 医疗条款解析与咨询 |
|------|-------------|-------------------|
| 输入 | 纯文本提问 | 文本 + 可选条款（粘贴/上传） |
| 知识来源 | 平台预设知识库 | 用户条款 + 平台知识库 |
| 上下文 | 对话历史 | 对话历史 + 当前条款 |
| 反问节点 | 要素缺失时 | 条款缺失且问题需条款支撑时 |

## 三、产品设计

### 3.1 用户旅程（三阶段）

```
Phase 1: 条款输入
  └── 方式 A：用户粘贴条款文本到输入框（可选，支持与问题一起发送）
  └── 方式 B：用户点击「上传条款」，选择 PDF/Word/TXT 文件
  └── 系统：解析/入库，建立「当前条款」会话上下文
  └── 提示："已加载您的条款，现在可以开始提问了。"

Phase 2: 多轮咨询
  └── 用户："免赔额怎么算？"
  └── 系统：结合「当前条款」+ 平台知识库，检索并生成答案
  └── 用户："住院前后门诊能报吗？"
  └── 系统：同上，持续基于当前条款回答

Phase 3: 输出与合规
  └── 答案需引用条款原文片段（如有）
  └── 结尾附合规声明："本解读仅供参考，具体以保险合同条款为准。"
```

### 3.2 输入方式设计

#### A. 文本输入（一期必做）

- **入口**：输入框支持「条款 + 问题」混合输入，或单独发送条款
- **识别规则**：若用户输入超过 200 字且含「条款」「合同」「保险责任」「免赔」等词，可推断为条款类内容
- **存储**：条款文本存入 Redis 上下文，key：`clause:{user_id}:{session_id}`，TTL 与对话一致

#### B. 文件上传（一期必做）

- **支持格式**：PDF、Word（.doc/.docx）、TXT
- **大小限制**：单文件 ≤ 10MB（可配置）
- **流程**：
  1. 用户上传 → 后端接收文件
  2. 创建/复用「用户条款知识库」（RAGflow Dataset）
  3. 调用 RAGflow 文档上传 API 入库
  4. 轮询解析状态，完成后将会话绑定该 Dataset ID
  5. 后续检索时 `dataset_ids = [默认库, 用户条款库]`

### 3.3 核心数据结构

#### 条款上下文（Redis）

```python
# Key: clause_ctx:{user_id}:{session_id}
# Value (JSON):
{
    "source": "text" | "upload",       # 来源
    "dataset_id": "xxx",              # 上传时 RAGflow 知识库 ID
    "text_preview": "前 500 字...",    # 文本时的预览
    "file_name": "某某医疗险条款.pdf",  # 上传时的文件名
    "created_at": 1739000000,
}
```

#### 意图 Slot（若需引导）

| 场景 | 是否需 Slot | 说明 |
|------|-------------|------|
| 用户已带条款（粘贴/上传） | 否 | 直接进入咨询 |
| 用户问「这段什么意思」但上下文无条款 | 是 | 追问："请粘贴相关条款内容，或上传条款文件。" |

### 3.4 反问策略

| 缺失项 | 反问示例 |
|--------|----------|
| 条款未提供且问题依赖条款 | "要准确解读，请先粘贴相关条款段落，或上传完整条款文件。" |
| 文件解析失败 | "文件解析失败，建议您复制条款文字粘贴到对话框，或换一个格式（如 TXT）重试。" |

### 3.5 UI 呈现建议

#### A. 输入区增强

- 输入框上方增加「上传条款」按钮（或拖拽区）
- 上传后显示：`已加载：xxx.pdf`，支持「清除」重新上传

#### B. 回复结构（沿用 Markdown）

```markdown
## 智保灵犀·条款解读

**您的问题：** [问题摘要]

**条款依据：** 
> [引用条款原文片段，如有]

**解读：**
[结构化解读内容]

---
本解读仅供参考，不构成法律或投保建议，具体以保险合同条款为准。
```

### 3.6 意图识别关键词

```python
INTENT_KEYWORDS[INTENT_CLAUSE_PARSE] = [
    "条款", "合同", "保险合同", "保险责任", "责任免除",
    "免赔额", "赔付比例", "等待期", "续保条件",
    "这段什么意思", "这条怎么理解", "能赔吗", "能不能报",
    "上传条款", "解析条款", "解读条款", "条款咨询",
]
```

**优先级**：若用户消息中同时带条款文本（>200 字）或刚上传文件，且含上述关键词，优先识别为 `clause_parse`。

## 四、技术方案概要

### 4.1 新增接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/clause/upload` | POST | 上传条款文件（multipart/form-data） |
| `/api/clause/context` | GET | 获取当前会话的条款上下文 |
| `/api/clause/clear` | POST | 清除当前会话的条款上下文 |

### 4.2 修改接口

| 接口 | 修改点 |
|------|--------|
| `POST /api/chat` | 请求体可选带 `clause_text`；响应可含 `clause_loaded` 标识 |
| `POST /api/chat/stream` | 同上，支持流式场景 |

### 4.3 配置项

```python
# 医疗条款解析
CLAUSE_PARSE_ENABLED: bool = True
CLAUSE_UPLOAD_MAX_SIZE: int = 10 * 1024 * 1024   # 10MB
CLAUSE_ALLOWED_EXT: str = "pdf,doc,docx,txt"
CLAUSE_KB_CHUNK_METHOD: str = "laws"             # RAGflow 分块：法律/条款类
```

## 五、验收标准

### 5.1 功能验收

| 项 | 预期 |
|----|------|
| 文本粘贴 | 用户粘贴条款后提问，能基于该条款回答 |
| 文件上传 | 上传 PDF/Word，解析成功后提问，能引用条款内容 |
| 多轮咨询 | 同一会话内无需重复上传，可持续提问 |
| 清除条款 | 用户清除后，后续问题不再依赖该条款 |
| 无条款提问 | 若问题需条款支撑但未提供，系统追问引导 |

### 5.2 非功能验收

| 项 | 预期 |
|----|------|
| 合规 | 答案含合规声明，不替代律师/客服建议 |
| 性能 | 文件解析在 30 秒内完成（视文件大小与 RAGflow 负载） |
| 兼容 | 与现有对话、流式、保障重叠等功能无冲突 |
