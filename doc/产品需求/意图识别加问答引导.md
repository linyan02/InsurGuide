### 第一部分：功能需求描述 (PRD)

**1. 需求背景**
在百万医疗险咨询场景中，用户提问通常非常模糊（如“那个报销住院的多少钱？”）。若直接进行 RAG 检索，往往因缺少关键核保要素（年龄、健康状况等）而召回无效信息。

**2. 核心目标**

* **语义对齐**：精准识别用户多样化的模糊表达，将其归类为“百万医疗险”意图。
* **智能引导**：当用户关键信息缺失时，中断检索，发起有温度的专业追问。
* **精准召回**：仅在要素补全后，执行基于“用户真实画像”的 RAG 检索。
* **富文本呈现**：输出包含条款解读、核保图片、理赔案例的结构化内容。

**3. 用户场景示例**

* **用户**：“我想买那个报销医疗费的。”
* **系统**（识别意图 -> 发现要素缺失）：分析出缺失年龄和健康状况，追问：“百万医疗险对年龄和健康要求较高。请问被保人多大年纪？过去两年是否有过住院记录？”
* **用户**：“我 45 岁，身体挺好。”
* **系统**（补全要素 -> RAG 检索）：输出针对 45 岁健康人群的医疗险对比、核保流程图及案例。

### 第二部分：产品技术方案文档

**1. 架构调整**
在 `services/rag/pipeline.py` 的执行流中，在意图识别与 RAG 检索之间插入**需求分析拦截器**。

**2. 关键逻辑组件**

* **意图泛化引擎 (`app/intent.py`)**：通过扩展语义概念集，捕捉涉及“住院报销”、“大额保障”、“核保”等主题的模糊提问。
* **要素矩阵 (`app/insurance_schemas.py`)**：定义百万医疗险的 6 大核心要素（年龄、医保、住院史、慢性病、器官结节、特殊需求）。
* **引导逻辑 (`app/llm_short.py`)**：利用 LLM 判定要素完整度。若不完整，生成包含“因果关联”解释的专业追问。
* **富文本引擎 (`app/answer_engine.py`)**：支持解析 RAGflow 返回的图片元数据，并将相似案例封装为 Markdown 引用块。

**3. 数据流向**

1. 用户输入 -> `intent.recognize` 判定为 `medical_insurance`。
2. 调用 `extract_insurance_slots` -> 结合上下文提取要素。
3. **判断**：
* `is_complete == False` -> 返回 `guide_question`，状态设为 `GUIDING`。
* `is_complete == True` -> 执行 `rewrite`（重组要素与问题） -> `call_ragflow` -> `generate_answer`。

### 验收清单

* **模糊识别**：输入“怎么报销医药费”，系统识别为医疗险。
* **智能拦截**：在未告知年龄和健康前，系统**不应**展示具体产品条款，而是发起追问。
* **数据闭环**：用户补充信息后，系统能结合前后文输出包含“图片”和“案例”的深度回答。
