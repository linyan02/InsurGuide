# 智保灵犀 - 功能增强建议（汇报导向）

> 基于当前实现完成度与拔高版 PPT 汇报的差距分析，提出的优先级功能建议  
> 生成日期：2026-02-27

---

## 一、现状差距概览

### 1.1 PPT 承诺 vs 实际完成度

| PPT / 产品方案声称 | 实际完成情况 | 差距 |
|-------------------|--------------|------|
| 用户上传 PDF 条款即可获得专属解读 | 后端 `/api/clause/upload` 已实现 | **前端未调用**：PC Web 有上传按钮，但文件从未发到后端，只发文本到 chat |
| 条款即传即问、会话级上下文 | clause_context、多库检索已实现 | **前端未传 session_id**：chat 请求无 session_id，无法绑定 clause_ctx |
| 条款粘贴 + 文件上传双入口 | 后端支持 `clause_text` 参数 | **前端无粘贴入口**：无「粘贴条款」折叠区，长文本无法作为 clause_text 传递 |
| 已加载 xxx.pdf、清除条款 | 后端 `/api/clause/context`、`/api/clause/clear` 已实现 | **前端无展示**：无条款状态栏、无清除按钮 |
| 可溯源、可审计、合规日志 | 已落库 interaction_logs、compliance_logs | **无管理端**：无法在汇报时展示「可回溯」看板 |
| 条款结构化抽取、多产品对比 | 产品三期规划 | **未实现**：PPT 未来规划中提及，可作为亮点提前做简化版 |

### 1.2 核心结论

**当前最大的断点在前端**：后端条款解析链路已打通，但 PC Web 未形成「上传→解析→多轮咨询」的闭环，汇报演示时无法展示「即传即问」的核心卖点。

## 二、功能增强建议（按优先级）

### P0 - 必做：补齐前端条款解析闭环（汇报可演示）

| 功能 | 说明 | 工作量 | 价值 |
|------|------|--------|------|
| **1. 文件上传对接 /api/clause/upload** | 用户选择 PDF/Word 后点击发送时：先调用 `POST /api/clause/upload`（multipart，需带 Authorization），成功后再调用 `POST /api/chat/stream` 并传入 `session_id` | 中 | **汇报核心演示**：真正实现「上传 PDF → 问免赔额」 |
| **2. session_id 生成与传递** | 每轮新对话生成 `session_id`（如 UUID），发 chat 时必传；侧栏「新对话」时重置 session_id | 小 | 条款上下文才能正确绑定会话 |
| **3. 条款状态展示** | 上传成功后显示「已加载：xxx.pdf」，并增加「清除条款」按钮，调用 `POST /api/clause/clear` | 小 | 用户感知、汇报展示「当前有条款上下文」 |
| **4. chat 请求增加 clause_text / session_id** | 当用户粘贴长文本（>200 字）且含条款关键词时，将文本作为 `clause_text` 传入；所有 chat 请求带 `session_id` | 小 | 支持「粘贴条款」场景 |

**预期效果**：汇报时可直接演示「上传条款 → 多轮追问免赔额、等待期」的完整流程。

---

### P1 - 高价值：强化汇报说服力

| 功能 | 说明 | 工作量 | 价值 |
|------|------|--------|------|
| **5. 条款粘贴入口** | 输入框上方增加「粘贴条款」可折叠区，用户粘贴后与问题一起发送，后端用 `clause_text` 接收 | 小 | 覆盖无 PDF、仅有文本的场景，PPT 双入口完整 |
| **6. 条款结构化抽取（简化版）** | 用户上传/粘贴条款后，首轮自动触发一次「抽取关键要素」：免赔额、等待期、续保条件等，以结构化卡片展示 | 中 | **PPT 差异化亮点**：「从解释者到规划师」的过渡，行业少见 |
| **7. 管理端基础看板（最小可行）** | 新增 `/admin` 或独立页面：展示近 7 天交互量、意图分布、合规拦截次数、条款解析调用次数 | 中 | **汇报「可回溯」佐证**：展示数据落库、可审计 |
| **8. 流式响应中展示 clause_loaded** | 前端解析 SSE 事件中的 `clause_loaded` 字段，在回答旁显示「基于您上传的条款」标识 | 小 | 用户感知、汇报时一眼看出「在用条款」 |

---

### P2 - 加分项：深化差异化

| 功能 | 说明 | 工作量 | 价值 |
|------|------|--------|------|
| **9. 多产品条款对比** | 用户上传 2 份条款（或选平台内 2 款产品），问「免赔额/续保条件有什么区别」，返回对比表格 | 大 | PPT 未来规划提前落地，决策支持升级 |
| **10. Gradio 条款解析入口** | Gradio 对话 Tab 增加「上传条款」按钮、条款状态、清除，与 PC Web 一致 | 中 | 多端一致、演示备用 |
| **11. 对话历史恢复时 clause 关联** | `context/restore` 恢复某条历史时，若该对话当时有条款上下文，一并恢复 clause_ctx（需 interaction_logs 扩展存储 clause 关联） | 中 | 体验完整，技术深度体现 |
| **12. 智能推荐问题** | 条款加载后，推荐「免赔额怎么算」「等待期多久」等 3～5 个快捷问题，点击即问 | 小 | 降低使用门槛，汇报体验更顺 |

## 三、推荐实施顺序（汇报前可完成范围）

### 方案 A：最小闭环（约 1～2 天）

仅做 **P0 的 1、2、3**：

- 前端：上传文件 → 先调 clause/upload → 再调 chat/stream 带 session_id
- 条款状态：「已加载 xxx.pdf」+ 清除按钮
- session_id 在新对话时生成

**汇报可演示**：「上传 PDF → 问免赔额」核心流程跑通。

---

### 方案 B：完整 P0 + 部分 P1（约 3～5 天）

在 A 基础上增加：

- P0-4：clause_text、session_id 在 chat 中传递
- P1-5：条款粘贴入口
- P1-8：流式响应展示 clause_loaded

**汇报可演示**：双入口（粘贴+上传）、条款状态、多轮咨询、可感知的「基于条款回答」。

---

### 方案 C：冲刺高分（约 1～2 周）

在 B 基础上增加：

- P1-6：条款结构化抽取（首轮自动抽取免赔额/等待期等，以卡片展示）
- P1-7：管理端基础看板
- P2-12：智能推荐问题

**汇报可演示**：结构化抽取 + 可回溯看板 + 智能引导，形成「技术深度 + 业务价值」双重支撑。

## 四、实现要点速查

### 4.1 前端条款上传对接（P0-1）

```javascript
// 伪代码：handleSend 中，当 hasFile 时
if (hasFile) {
  const fileInput = document.getElementById('f-upload');
  const file = fileInput?.files?.[0];
  if (file) {
    const formData = new FormData();
    formData.append('file', file);
    formData.append('session_id', getSessionId());
    const uploadRes = await fetch(API_BASE + '/api/clause/upload', {
      method: 'POST',
      headers: { 'Authorization': 'Bearer ' + getToken() },
      body: formData,
    });
    if (!uploadRes.ok) { /* 错误处理 */ return; }
  }
}
// 然后调用 chat/stream，body 中加 session_id: getSessionId()
```

### 4.2 session_id 生成

- 新对话：`session_id = 'sess_' + Date.now() + '_' + Math.random().toString(36).slice(2)`
- 侧栏「新对话」：重置为新的 session_id
- 恢复历史：可选沿用原 session_id（若后端支持 clause 关联恢复）

### 4.3 条款结构化抽取（P1-6）简化实现

- 条款加载后，调用一次专用接口或 chat 时带 `extract_structured: true`
- 后端用 LLM 抽取：免赔额、等待期、续保条件、责任免除要点
- 返回 JSON，前端渲染为卡片（如「免赔额：1 万元」「等待期：30 天」）

## 五、小结

| 维度 | 建议 |
|------|------|
| **汇报前必做** | P0：前端条款上传闭环、session_id、条款状态、chat 传 clause_text |
| **高分加成** | P1：条款粘贴、结构化抽取、管理端看板、clause_loaded 展示 |
| **差异化冲刺** | P2：多产品对比、智能推荐问题 |
| **实施节奏** | 先完成方案 A 保证可演示，再按时间择机做 B/C |

当前后端能力已就绪，**补齐前端闭环是见效最快、汇报收益最高的投入**。
