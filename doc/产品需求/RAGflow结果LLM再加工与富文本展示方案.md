# RAGflow 结果 LLM 再加工与富文本展示 —— 产品技术方案

## 一、需求背景与目标

### 1.1 现状
- **RAGflow 检索**：返回知识库片段（documents + metadatas）
- **答案生成**：当前由 `answer_engine.py` 一次性将检索结果 + 上下文输入 LLM 生成答案
- **前端展示**：仅当答案包含 `![`、`> **相似案例**`、`\n> ` 等特定模式时才用 Markdown 渲染，否则使用 `nl2br` 纯文本换行

### 1.2 核心诉求
1. **LLM 再加工**：RAGflow 结果需经大模型进一步加工，形成**清晰、结构化**的意见输出
2. **便于理解**：用户能快速把握核心结论、细节说明与注意事项
3. **富文本支持**：页面回答需支持富文本格式（标题、列表、加粗、引用、表格等）

### 1.3 产品目标
- 提升回答可读性与专业感
- 统一输出结构，降低用户认知负担
- 兼容保险行业合规要求（引用来源、免责声明）

---

## 二、产品设计

### 2.1 输出内容结构（结构化意见模板）

| 模块 | 说明 | 示例 |
|------|------|------|
| **核心结论** | 1-2 句简明回答 | 蓝医保长期医疗险支持甲状腺结节承保，需人工核保且可能除外。 |
| **详细说明** | 分点展开，引用条款 | • 投保需如实告知健康异常；<br>• 核保结果以保险公司为准... |
| **注意事项** | 提醒与风险点 | ⚠️ 既往症通常不赔；续保需满足健康告知... |
| **参考来源** | 知识库文档溯源 | 来源：蓝医保条款_投保规则.txt |
| **合规声明** | 固定尾注 | 本内容仅供参考，不构成投保建议，具体以保险合同条款为准。 |

### 2.2 输出格式规范（Markdown 富文本）

统一要求 LLM 输出**标准 Markdown**，便于前端渲染：

- **标题**：`## 核心结论`、`### 详细说明`
- **列表**：`-` 或 `1.` 有序列表
- **强调**：`**重点内容**`、`*斜体*`
- **引用**：`> 条款原文引用`
- **分割**：`---` 分隔不同模块
- **表格**（可选）：产品对比、保障项对比

### 2.3 用户场景与体验

| 场景 | 用户期望 | 方案应对 |
|------|----------|----------|
| 条款查询 | 快速知道结论与出处 | 核心结论 + 参考来源 |
| 产品对比 | 结构化对比项 | 表格或分点列表 |
| 理赔指引 | 步骤清晰、可执行 | 有序列表 + 注意事项 |
| 免责说明 | 风险点突出 | 注意事项 + 合规声明 |

### 3.2 方案选型：单阶段 vs 两阶段 LLM

| 方案 | 描述 | 优点 | 缺点 | 推荐 |
|------|------|------|------|------|
| **A. 单阶段强化** | 优化现有 Prompt，要求一次生成结构化 Markdown | 实现简单、时延低、成本低 | 长回答时格式可能不稳定 | ✅ 一期首选 |
| **B. 两阶段** | 首轮生成初稿 → 第二轮精修为结构化意见 | 格式更稳定、可做二次校验 | 双倍调用、时延与成本升高 | 二期可选 |

**一期采用方案 A**：通过精心设计的 Prompt 模板 + Few-shot 示例，引导单次生成结构化输出；若效果不足再考虑两阶段。

## 四、详细设计

### 4.1 后端： answer_engine 改造

#### 4.1.1 新增「结构化意见」Prompt 模板

```python
# 新增 INSURANCE_OPINION_PROMPT_TEMPLATE
INSURANCE_OPINION_PROMPT_TEMPLATE = """
你是一位专业的保险顾问。请根据以下知识库内容，针对用户问题输出【结构化意见】。

## 输出要求（必须使用 Markdown 格式）：
1. **## 核心结论**：1-2 句简明回答，直接回应问题。
2. **## 详细说明**：分点展开（使用 - 或 1. 2. ），引用知识库关键信息，禁止编造。
3. **## 注意事项**：提醒用户需关注的风险点、除外情形或续保要求（如有）。
4. **合规声明**：结尾必须添加：本内容仅供参考，不构成投保建议，具体以保险合同条款为准。

## 格式规范：
- 使用标准 Markdown：标题用 ##、###，列表用 - 或数字，强调用 **加粗**
- 引用条款时用 > 引用块
- 语言简洁、专业，避免术语堆砌

## 知识库内容：
{knowledge_content}

## 历史上下文：
{context}

## 用户问题：
{query}

请直接输出结构化意见（Markdown），不要输出其他前缀或说明。
"""
```

#### 4.1.2 配置开关

- 新增配置项 `ANSWER_USE_OPINION_FORMAT`（默认 `True`）
- 兼容旧版：`False` 时沿用原 `INSURANCE_PROMPT_TEMPLATE`

#### 4.1.3 流程调整

1. `generate_answer()` 根据配置选择 Prompt 模板
2. 保持现有 `enrich_answer_with_rich_content()`：图片、案例仍追加到答案末尾
3. 合规校验在答案生成后统一执行，不改变流程
4. 输出统一为 Markdown 字符串，与现有 API 字段 `answer` 一致

#### 4.1.4 LangChain 模式

- `langchain_chain.py` 中同步替换 `INSURANCE_PROMPT_TEMPLATE` 为结构化版本
- 保证两种 RAG 模式输出格式一致

### 4.2 前端：富文本展示改造

#### 4.2.1 渲染策略（统一 Markdown）

**当前逻辑**（`web/static/index.html`）：

```javascript
// 仅当存在特定模式时才 Markdown，否则 nl2br
if (rawAnswer.includes('![') || rawAnswer.includes('> **相似案例**') || rawAnswer.includes('\n> ')) {
    html = marked.parse(rawAnswer);
} else {
    html = nl2br(rawAnswer);
}
```

**改造后逻辑**：

- **统一使用 marked 渲染**：所有 `answer` 均按 Markdown 解析
- 保留 `breaks: true`，兼容换行
- 对空内容或纯空格做安全过滤，避免 XSS

```javascript
// 伪代码
function renderAnswer(rawAnswer) {
    if (!rawAnswer || !rawAnswer.trim()) return '';
    try {
        marked.setOptions({ breaks: true });
        let html = marked.parse(rawAnswer);
        html = html.replace(/<blockquote>/g, '<blockquote class="case-card">');
        return html;
    } catch (e) {
        return nl2br(escapeHtml(rawAnswer)); // 降级为安全纯文本
    }
}
```

#### 4.2.2 样式增强

- 标题层级：`##` → `h2`，`###` → `h3`，保证层级清晰
- 列表：统一缩进、行距
- 引用块：沿用 `.case-card` 样式（左侧色条、浅蓝背景）
- 表格：`<table>` 基础样式（边框、斑马纹）

#### 4.2.3 历史记录恢复

- `get_chat_history_item` 返回的 `answer` 已是 Markdown
- 恢复展示时同样使用统一 Markdown 渲染，逻辑与实时对话一致

---

### 4.3 API 约定

**响应结构不变**，`answer` 字段改为「结构化 Markdown 文本」：

```json
{
  "code": 200,
  "data": {
    "answer": "## 核心结论\n\n蓝医保支持甲状腺结节承保...",
    "source": ["蓝医保条款_投保规则.txt"],
    "intent_cn": "百万医疗险",
    "state": "COMPLETE",
    ...
  }
}
```

- 不新增字段，避免前端大范围改动
- 如需区分「旧版纯文本」与「新版 Markdown」，可增加 `answer_format: "markdown"`（可选）

---

### 4.4 Gradio UI

- `gradio_ui/pages/chat.py` 中 `rag_answer` 使用 `gr.Markdown` 替代 `gr.Textbox`，直接渲染富文本
- 保证 Gradio 演示与 Web 端展示一致

## 六、效果验收

### 6.1 验收标准

## 七、附录

### A. Few-shot 示例（可选注入 Prompt）

```
示例问题：蓝医保甲状腺结节能买吗？
示例输出：
## 核心结论
蓝医保长期医疗险支持甲状腺结节人群投保，需进行人工核保，核保结果可能为标准体、除外承保或拒保。

## 详细说明
- 投保时需在健康告知中如实填写甲状腺相关检查异常
- 保险公司将根据结节大小、分级等进行人工核保
- 核保结果以保险公司出具的核保结论为准

## 注意事项
既往症（投保前已有疾病）通常不在保障范围内；续保时需满足健康告知要求。

---
本内容仅供参考，不构成投保建议，具体以保险合同条款为准。
```
